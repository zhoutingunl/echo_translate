# EchoTranslate · 5 分钟演示脚本

目标：在 5 分钟内清晰展示「实时字幕 → 自动纠错 → 术语 → 模式 → 语音 → 纪要 → QoS」。

## 准备
```bash
cp .env.example .env      # 填 MINIMAX_API_KEY
python app.py
```
浏览器（Chrome）开两个标签：`http://127.0.0.1:8000` 与 `http://127.0.0.1:8000/dashboard`。

## 分镜（约 5 分钟）

**① 开场（30s）** 一句话点题：用户的痛点是「跟不上语速」而非「不懂单词」；本作把英文音频实时转成中文字幕/语音，并能自动纠错。

**② 自动纠错 —— 核心亮点（90s）**
- 字幕页选「技术分享：分布式存储系统」，点 **▶️ 播放示例**。
- 边播边讲：每句英文落地即出中文（增量翻译）。
- 重点指出两处 `修正` 高亮：
  - 「存储系统」→「**分布式**存储系统」（对应 ASR 把 storage 修订为 distributed storage）；
  - 「缓存层」→「**分布式**缓存层」。
  - 强调：旧译文以删除线短暂保留，最近 5 秒内的字幕才可回滚——这正是 *Revision Window*。

**③ 术语记忆（45s）** 指出 Kafka / Redis / Kubernetes 全程保持原样、未被音译。侧栏「术语表」现场加一个词（如 `Pulsar`），说明新增术语会回溯重渲染窗口内字幕。

**④ 实时聆听 + 双模式 + TTS（60s）** 点 **🎙️ 开始聆听**，对麦克风念一段英文（或播一段英文视频）展示真·实时；切「会议模式（低延迟）」对比风格；勾选 **🔊 中文语音播报** 让它读出译文。

**⑤ 纪要（20s）** 点 **📝 生成纪要**，展示 AI 把整场压缩成要点。

**⑥ QoS Dashboard（45s）** 切到 `/dashboard`：字幕延迟 avg/P95/P99、RTF、翻译成功率、术语命中率、纠错次数，逐项对照目标值显示「✅ 达标」。强调指标来自真实埋点，非硬编码。

**⑦ 收尾（20s）** 一句话讲取舍：AI 预算集中在翻译+纠错，ASR/TTS 用浏览器原生换零延迟；删掉了实测无效的「伪纠错」，只保留真正改变输出的纠错路径。

## 备用（无网络/无麦克风）
全程用「示例回放」即可完成 ②③⑤⑥；回放只替换原文来源，翻译与纠错仍是真实 AI 实时产生。
